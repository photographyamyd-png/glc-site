WHAT A "SECTION TEMPLATE" IS HERE
---------------------------------
A reusable React component (usually under components/glc-sections/ or
components/ground-level/) that owns STRUCTURE and VISUAL SHELL for a recurring
band: e.g. full-bleed image, scrims, z-order, max-width container, optional
min-height floor. COPY, heading levels, CTAs, and split layouts live in
children or typed props you pass from a page or components/templates/* file.

It is NOT a separate file format. You refactor by replacing an inline
<section> block with an import + props/children while preserving semantics,
tonal neighbors, and SEO.

STEPS — REFACTOR AN EXISTING SECTION TO THIS TEMPLATE (flexible, safe)
--------------------------------------------------------------------------------
1) LOCATE
   - File path + approximate line range, or unique string in the section.
   - Note current: band class (band-dark / band-dark-field), image src/alt,
     scrim classNames, inner max-width, padding, heading id, section id.

2) PRESERVE SEMANTICS
   - Keep the same heading LEVEL (h1/h2/h3) unless the page IA is intentionally
     changing. One H1 per page view where applicable (seo-strategy.mdc).
   - Keep stable id= and aria-labelledby= (or add if missing).

3) SPLIT SHELL VS CONTENT
   - Shell = section wrapper, Image fill, scrims, z-10 outer flex, optional
     min-h-* on OUTER section only as a FLOOR (e.g. min-h-[560px]); inner uses
     flex so content grows — no fixed pixel heights for the text block.
   - Content = eyebrow, titles, paragraphs, quotes, buttons — passed as
     children (or small typed slots). If body grows past ~2 sentences, apply
     copy-writing.mdc: split column / asymmetric layout INSIDE children, not
     by shrinking type alone.

4) SPACING RHYTHM (modular, not arbitrary)
   - Keep ratios consistent: e.g. heading-to-body gap vs card padding should
     follow existing tokens (--s7, mt-6, mt-8, py-20, etc.) already used in
     siblings or design-system.mdc spacing guidance.
   - Heavy text (> ~4 lines): increase line-height / measure (max-w-prose,
     leading-[1.72]) and add vertical whitespace — do not cram.

5) DESIGN RULES
   - No new random colors or one-off RGB unless matching an existing approved
     scrim from the same pattern elsewhere in the repo.
   - Tonal alternation: do not place two same-tone major sections back-to-back
     without a seam or intentional exception (design-system.mdc).
   - Tri-tone type roles on dark: white / white-90 / yellow accent; eyebrows
     per .eyebrow rules.

6) VERIFY
   - Visual parity OR intentional delta (state which).
   - Keyboard / focus on CTAs; min tap 44px where interactive.
   - Reduced motion: respect prefers-reduced-motion on motion-heavy bits.
